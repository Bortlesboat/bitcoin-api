x402 for AI agents and premium Bitcoin API calls

Satoshi API uses x402 as the pay-per-call lane for premium endpoints. An agent can request a higher-value route, receive a 402 Payment Required challenge, pay in USDC on Base, and retry without a signup flow.

Why x402 exists here

Most of Satoshi API stays free. x402 is reserved for the endpoints where the cost is real or the value is meaningfully higher, such as AI analysis, transaction broadcast, next-block prediction, and fee observatory analytics.

Why agents care

x402 keeps payment inside the same request-response loop. That matters when an agent needs one premium result right now and should not stop for signup, billing setup, or a copy-pasted API key.

How it fits the product

Satoshi API uses a hybrid model: free public Bitcoin data, API keys for recurring usage, and x402 for keyless pay-per-call access. That is the lane this page documents.

The x402 flow on Satoshi API

  1. 1
    Call a premium endpoint.

    The API responds with a machine-readable 402 Payment Required payload.

  2. 2
    Read the payment requirements.

    The response tells the client what asset, network, and amount are required.

  3. 3
    Pay and retry.

    The client signs the payment on Base and resends the request with an X-PAYMENT header.

  4. 4
    Receive the premium response.

    Once the payment is verified, the endpoint returns the requested data.

Try the demo flow

Use the demo endpoint to see the exact 402 challenge shape without spending real funds.

# Step 1: request the challenge
curl https://bitcoinsapi.com/api/v1/x402-demo

# Step 2: retry with the demo payment header
curl -H "X-PAYMENT: demo" \
  https://bitcoinsapi.com/api/v1/x402-demo

What stays free?

The public fee, block, mempool, network, price, history, and discovery surfaces remain open. x402 only gates the premium routes listed below.

Why not just issue a key?

API keys are still the best fit for recurring usage. x402 is for the moments where a one-off premium result should be paid for inline instead of forcing account setup first.

Does this work with MCP?

Yes. Agents can discover the free API and MCP surface first, then use x402 for premium calls when the workflow justifies the extra step.

x402 on Satoshi API is still early. Most activity today is challenge testing rather than completed payments. Use the demo endpoint to walk the flow.
